## 2. 项目目标

构建一个面向小学三年级学生的英语口语对练 Agent，支持：

- **按课程对练**：跟随导入的三年级语音教材内容进行跟读、对话练习
- **自由对练**：与 AI 进行开放式英语对话
- **记忆功能**：记录用户学习进度、薄弱环节、偏好等
- **手机端 UI**：移动端友好的 Web 界面

### 4.2 DDL (建表语句摘要)

```sql
-- 1. 课程表
CREATE TABLE courses (
    id INT AUTO_INCREMENT PRIMARY KEY,
    grade VARCHAR(20) NOT NULL COMMENT '年级，如 Grade3',
    semester VARCHAR(20) NOT NULL COMMENT '学期，如 上/下',
    version VARCHAR(50) DEFAULT '2024' COMMENT '教材版本',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) COMMENT '课程主表';

-- 2. 课程单元表
CREATE TABLE course_units (
    id INT AUTO_INCREMENT PRIMARY KEY,
    course_id INT NOT NULL,
    unit_name VARCHAR(100) NOT NULL COMMENT '单元名称，如 Unit1 Hello',
    unit_order INT NOT NULL COMMENT '单元序号',
    description TEXT COMMENT '单元描述',
    FOREIGN KEY (course_id) REFERENCES courses(id)
) COMMENT '课程单元';

-- 3. 句子/对话表
CREATE TABLE sentences (
    id INT AUTO_INCREMENT PRIMARY KEY,
    unit_id INT NOT NULL,
    sentence_text TEXT NOT NULL COMMENT '英文句子',
    sentence_order INT NOT NULL COMMENT '排序',
    audio_url VARCHAR(500) COMMENT '标准发音音频URL',
    translation VARCHAR(500) COMMENT '中文翻译',
    difficulty ENUM('easy','medium','hard') DEFAULT 'easy',
    FOREIGN KEY (unit_id) REFERENCES course_units(id)
) COMMENT '课程句子';

-- 4. 对话历史表
CREATE TABLE conversations (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL DEFAULT 1 COMMENT 'Demo阶段固定用户',
    session_id VARCHAR(64) NOT NULL COMMENT '会话ID',
    mode ENUM('course','free_talk') NOT NULL,
    course_unit_id INT DEFAULT NULL,
    role ENUM('user','assistant','system') NOT NULL,
    message_text TEXT NOT NULL,
    audio_url VARCHAR(500) DEFAULT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_session (user_id, session_id, created_at)
) COMMENT '对话记录';

-- 5. 用户学习进度
CREATE TABLE user_progress (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL DEFAULT 1,
    sentence_id INT NOT NULL,
    practice_count INT DEFAULT 0,
    score_avg FLOAT DEFAULT 0 COMMENT '平均评分',
    last_practice_at TIMESTAMP NULL,
    mastered BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (sentence_id) REFERENCES sentences(id),
    UNIQUE KEY uk_user_sentence (user_id, sentence_id)
) COMMENT '用户进度';

-- 6. 记忆记录表
CREATE TABLE memory_records (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL DEFAULT 1,
    memory_type ENUM('vocab_mastered','vocab_weak',
                     'grammar_weak','pronunciation_issue',
                     'learning_preference','interest_topic') NOT NULL,
    `key` VARCHAR(200) NOT NULL COMMENT '记忆关键词',
    `value` TEXT COMMENT '记忆内容',
    confidence FLOAT DEFAULT 0.5 COMMENT '置信度',
    last_updated TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_type (user_id, memory_type)
) COMMENT '用户记忆';
```

## 5. 核心功能模块

### 5.1 课程对练模式

```
流程:
  选择课程 → 选择单元 → 逐句练习 → 评分反馈 → 记录进度

详细步骤:
  1. 展示课程树 (年级 → 单元)
  2. 选中单元后，按顺序展示句子
  3. 播放标准发音 (TTS)
  4. 用户跟读录音 (ASR)
  5. 评分: 对比标准发音，给出音准/流利度/完整度分数
  6. 记录练习结果到 user_progress
  7. 根据薄弱点更新 memory_records
```

### 5.2 自由对练模式

```
流程:
  开始对话 → ASR转文字 → LLM回复 → TTS播放 → 存储记忆

详细步骤:
  1. 用户点击"开始对话"
  2. 系统根据 memory_records 构建 System Prompt (含已知词汇、薄弱点、偏好话题)
  3. 用户说话 → ASR 转文字
  4. 文字 + 历史对话 → 发送给 LLM
  5. LLM 返回适合小学生的英语回复 → TTS 播放
  6. 分析对话内容，提取新记忆点 (新学词汇、发音问题等)
  7. 存储对话和记忆
```

### 5.3 记忆功能设计

记忆系统采用**结构化记忆 + 动态提示注入**方式：

| 记忆类型 | 说明 | 用途 |
|----------|------|------|
| `vocab_mastered` | 已掌握词汇 | 避免重复练习，提升对话难度 |
| `vocab_weak` | 薄弱词汇 | 在对话中重点复习 |
| `grammar_weak` | 薄弱语法点 | 引导式纠正 |
| `pronunciation_issue` | 发音问题 | 针对性纠音 |
| `learning_preference` | 学习偏好 | 调整教学风格 |
| `interest_topic` | 兴趣话题 | 自由对话话题引导 |

**记忆注入方式**：
```
每次对话/课程开始时，查询用户记忆 → 构建动态 System Prompt:

"You are an English teacher for a 3rd-grade student.
Student profile:
- Mastered words: [apple, cat, dog, hello, ...]
- Weak words: [elephant, beautiful, ...]
- Pronunciation issues: [th sound, r/l confusion]
- Interests: [animals, sports]
Please adapt your teaching accordingly."
```

## 10. 关键设计决策

### 10.1 Demo 阶段简化

- **单用户**：user_id 固定为 1，不做登录/注册
- **不使用 Redis**：Demo 阶段直接查 MySQL，后续可引入缓存
- **不使用 WebSocket**：Demo 阶段用 HTTP 轮询替代实时流式

### 10.2 语音处理方案

```
录音 → WebM/WAV → 后端 ASR → 文字 → LLM → 文字 → TTS → 音频 → 前端播放

ASR: 腾讯云实时语音识别 (有儿童英语优化)
TTS: Edge-TTS (免费，音质可接受) 或 腾讯云 TTS
```

### 10.3 LLM Prompt 设计原则

- **角色设定**：你是一个耐心、鼓励性的小学英语老师
- **语言难度**：使用三年级学生能理解的简单英语
- **纠错策略**：不直接指出错误，而是用正确的表达重复一遍
- **激励反馈**：每次对话给予积极正面的鼓励

### 10.4 评分机制

| 维度 | 权重 | 说明 |
|------|------|------|
| 音准 (Accuracy) | 40% | 单词发音准确性 |
| 流利度 (Fluency) | 30% | 语速和停顿自然度 |
| 完整度 (Completeness) | 30% | 是否完整说出句子 |
